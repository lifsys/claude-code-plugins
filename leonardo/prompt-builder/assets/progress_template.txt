## Quick Reference

### File Structure
```
project/
├── src/           # Frontend source
├── server/        # Backend source
├── tests/         # Test files
├── features.json  # Feature tracking
├── init.sh        # Setup script
└── progress.txt   # This file
```

### Important Files
- `features.json` - Feature list with tests (DO NOT MODIFY TESTS)
- `init.sh` - Environment setup and verification
- `.env` - Environment variables (NEVER commit)
